1. Project Overview

The Xed 256 Official Website is a production-ready artist portfolio and e-commerce platform built for Edward Muligirwa, professionally known as Xed 256 — a genre-defying Ugandan musician from Fort Portal.

The website serves as the artist's digital home, combining music streaming, live event ticketing, merchandise sales, press resources, and fan engagement into a unified, visually stunning experience under the brand tagline "FortPortal Ni Dubai."

Key Objectives

  • Showcase music — stream all 9 tracks directly via YouTube integration
  • Build fanbase — blog, social links, newsletter, Linktree-style links page
  • Sell merchandise — full e-commerce with cart, multi-payment checkout
  • Sell event tickets — tiered ticketing (Regular/VIP/VVIP) with PDF ticket generation
  • Professional presence — press kit, interview features, booking form

8. E-Commerce & Payments

Cart System

The shopping cart is managed via React Context (lib/cart-context.tsx) providing global state across all pages:

  • addItem(product, size?) — Adds product to cart, auto-opens drawer
  • removeItem(productId, size?) — Removes specific item
  • updateQuantity(productId, quantity, size?) — Updates count, removes if 0
  • clearCart() — Empties entire cart
  • Currency toggle — Switch between USD and UGX site-wide
  • Computed totals — totalItems and totalPrice auto-calculated based on currency

Checkout Flow (3 Steps)

StepFieldsValidation
1. Order ReviewItem list, quantities, currency toggleAt least 1 item required
2. ShippingFirst/last name, email, phone, address, city, country, ZIPRequired fields, email format. Skipped for digital-only orders.
3. PaymentPayment method selector + method-specific formMethod-specific validation (card number, phone, reference)

9. Event Ticketing System

Ticket Purchase Flow

Located at /events/tickets/[eventId], the ticketing system provides a complete purchase experience:

StepWhat Happens
1. Select TicketChoose tier (Regular/VIP/VVIP) with pricing, perks list, remaining count. Select quantity (1–10). Toggle USD/UGX.
2. Your DetailsEnter name, email, phone. Ticket will be sent to this email.
3. PaymentChoose from all 6 payment methods. Fill method-specific form. Submit payment.
4. ConfirmationOrder ID generated. "Download Ticket PDF" button. Email simulation logged.

PDF Ticket Generation

The lib/generate-ticket-pdf.ts module generates a professionally designed HTML ticket that opens in a new browser tab for printing or saving as PDF. The ticket includes:

  • XED 256 gold gradient branding and logo
  • Event name, date, time, venue, and city
  • Ticket tier with perks list
  • Buyer name, email, and phone
  • Unique order ID (e.g., XED-TKT-M1A2B3-X4Y5)
  • QR code placeholder for venue scanning
  • Total amount paid with payment method
  • Terms and contact information

11. SEO & Structured Data

Metadata

Global metadata is configured in app/layout.tsx with per-page overrides:

  • Title template: %s | Xed 256
  • metadataBase: https://xed256.com
  • Open Graph: type, locale, siteName, title, description, image (1200×630)
  • Twitter Card: summary_large_image with @xed256 creator
  • Keywords: 13 targeted keywords including local Uganda/Fort Portal terms
  • Robots: index: true, follow: true

Sitemap (/sitemap.xml)

Auto-generated via app/sitemap.ts covering all static pages, product pages, and blog posts with appropriate changeFrequency and priority values.

Robots (/robots.txt)

Allows all crawlers. Disallows /checkout and /api. Points to sitemap URL.

Key Mobile Adaptations

  • Buttons: px-5 sm:px-8 py-2.5 sm:py-3 text-sm sm:text-base — fit 320px screens
  • Hero headings: text-3xl xs:text-4xl sm:text-6xl — scale from mobile to desktop
  • CTAs: Stack vertically on mobile (flex-col xs:flex-row), full-width buttons
  • Product grids: grid-cols-1 xs:grid-cols-2 md:grid-cols-3 lg:grid-cols-4
  • Decorative elements: Hidden on mobile (hidden sm:block) to save CPU/GPU
  • Sticky sidebars: Only sticky on desktop (lg:sticky lg:top-28)
  • Touch targets: Minimum 44×44px on all interactive elements
  • Filter tabs: Horizontal scroll on mobile (overflow-x-auto)
  • Cart drawer: Reduced padding and image sizes on mobile
  • Footer credit: Line breaks on mobile via <br className="sm:hidden">

15. Setup & Deployment

Prerequisites

  • Node.js 18+ installed
  • npm or yarn package manager
  • Git (for version control)

Quick Start

# Clone the repository
git clone git@github.com:Bahindiemma/xed256-official.git
cd xed256-official

# Install dependencies
npm install

# Start development server
npm run dev

# Open in browser
http://localhost:3000

Build for Production

# Create optimized production build
npm run build

# Start production server
npm start

Deploy to Vercel

# One-command deployment
npx vercel

Environment

No environment variables are required for the current static implementation. Future integrations (Stripe, email service, CMS) would use .env.local.
